**Game Design Document v2.0 — 웹 버전 완전 동일 기능 기준**

## 2. 화면 레이아웃

**모델: 풀스크린 게임 월드 + 카메라 follow + 반투명 오버레이 1레이어.**
HUD/D-Pad/Skill 영역이 게임 화면을 잘라내지 않는다. 모두 게임 위에 반투명으로 떠 있다.

```
┌──────────────────────────────────────────────────────┐
│  ⏱ 00:42  🎵 12점  🔥 3콤보  [SKILL: 22s]           │  ← HUD (반투명, 카메라 좌표계)
│        ♪                                             │
│                       👵 ← 수간호사 (월드 좌표계)    │
│                        F↓                            │
│                                                      │
│                  🧑‍⚕️ ← 김간호 (항상 화면 중앙)       │
│                                                      │
│                     ♪                                │
│  [SKILL]                            [▲]              │
│  (반투명)                       [◀][●][▶]  ← D-Pad   │
│                                     [▼]   (반투명)   │
└──────────────────────────────────────────────────────┘
        └─ 게임 월드 = 화면 풀스크린 ─┘
        └─ 맵(32×20타일=640×400)은 화면보다 큼 ─┘
        └─ 카메라(SKCameraNode)가 캐릭터 추적 ─┘
```

### 카메라 / 월드 모델
- 모든 게임 노드(맵·캐릭터·음표·적·투사체)는 `worldNode: SKNode` 자식.
- `cameraNode: SKCameraNode`가 매 프레임 `player.position`을 추적 (필요 시 lerp으로 부드럽게).
- 카메라는 월드 경계 밖으로 나가지 않게 클램핑 (양 끝에서는 캐릭터가 화면 중앙을 벗어남).
- HUD/D-Pad/Skill은 `cameraNode`의 자식 → 항상 화면에 고정되어 보임.

### HUD 구성 요소 (카메라 좌표계, 반투명)
- **Time**: 남은 시간(초). 10초 이하 → 빨간색 강조
- **Score**: 현재 점수
- **Combo**: 현재 콤보 수 (콤보 3 이상 시 강조)
- **Best**: 이번 세션 최고 기록
- **Skill**: 스킬 쿨다운 링 + 스킬명 표시 (스킬 없는 캐릭터는 `—`)

### 조작 패드 (카메라 좌표계, 반투명)
- **왼쪽 하단**: 스킬 버튼 (캐릭터별 능동 스킬 발동)
- **오른쪽 하단**: D-Pad 4방향 버튼 (누르고 있는 동안 이동)
- 투명도: 알파 0.3 (게임 화면이 가려지지 않도록)
- `zPosition`: HUD/오버레이 모두 게임 노드보다 위

## 3. 화면 흐름 (오버레이 순서)

```
앱 실행
  └─ [시작 오버레이]
       - 게임 제목, 스토리 설명
       - 난이도 선택 (하/중/상)
       - 목표 점수 표시
       - "시작" 버튼
            ↓
     [캐릭터 선택 오버레이]
       - 5명 캐릭터 카드 (이름, 태그, 픽셀 아바타)
       - "← 난이도 다시" / "이 친구로 시작" 버튼
            ↓
     [스킬 설명 오버레이]  ← 스킬 없는 김간호 선택 시 스킵
       - 캐릭터 아바타
       - 스킬명 + 설명 + 쿨다운 + 조작 키 안내
       - "← 캐릭터 다시" / "시작" 버튼
            ↓
     [인트로 컷씬]  ← 난이도별 텍스트 분기
            ↓
     [석조무사 경고 컷씬]  ← 하/중 난이도에서 게임 시작 전
     [이교수 경고 컷씬]  ← 상 난이도에서 게임 시작 전
            ↓
     [게임 플레이 — 45초]
       ├─ 15초 경과 → [중간 컷씬 1: 캐릭터 속마음]
       ├─ 30초 경과 → [중간 컷씬 2: 수간호사 경고]
       └─ F 피격 or 타임업
            ↓
     [종료 오버레이]
       - 결과: 이번 점수, 최고 기록 갱신 여부
       - 통계: 최대 콤보, F 피격 횟수, 정확도
       - 내 캐릭터 최고 기록 (하/중/상)
       - 다른 캐릭터 기록 보기 토글
       - 엔딩 스토리 텍스트
       - 조건 달성 시: "HG가 실습 때 만든 노래 듣기" 버튼
       - 3난이도 목표 달성 시: "졸업장 보기" 버튼
       - "다시 플레이" / "난이도 다시 선택" / "타이틀로" 버튼
            ↓ (조건 달성 시)
     [졸업장 오버레이]
       - 실습 수료 증서 (영/한 제목)
       - 캐릭터 픽셀 아바타
       - 졸업 본문 텍스트 + 날짜 + 발급자
       - "이미지로 다운로드" / "닫기" 버튼
```

## 4. 캐릭터 시스템 (5명)

능력치는 전원 동일. 외형·이름·스킬만 차별화.

| ID | 이름 | 태그 | 스킬 |
|---|---|---|---|
| `kim` | 김간호 | 번머리 실습생 | **없음** |
| `jung` | 정간호 | 곡괭이 근육 | 암벽등반 돌진 |
| `geon` | 건간호 | 안경과 책 | 북클럽 소집 |
| `im` | 임간호 | 긴머리 냥 | 나는야 모범생 |
| `lee` | 이간호 | 단발 댕댕 | 대만여행 |

### 스킬 상세

| 캐릭터 | 스킬명 | 효과 | 지속 | 쿨다운 |
|---|---|---|---|---|
| 정간호 | 암벽등반 돌진 | 바라보는 방향으로 3타일 돌진, 벽 1칸 파괴 | 260ms | 22초 |
| 건간호 | 북클럽 소집 | 주변 6타일 안 음표를 한 번에 끌어와 수집 | 즉발 | 20초 |
| 임간호 | 나는야 모범생 | 수간호사를 매혹 → F 대신 A 투척. A 수집 시 점수 2배 | 1500ms | **게임당 1회** |
| 이간호 | 대만여행 | 가장 먼 빈 타일로 순간이동, 착지 후 0.5초 무적 | 500ms | 22초 |

- 조작: D-Pad 왼쪽 스킬 버튼 (웹: Shift 키)
- 쿨다운 중 HUD에 링 형태로 남은 시간 표시

## 5. 난이도 시스템 (3단계)

| 항목 | 하(easy) | 중(normal) | 상(hard) |
|---|---|---|---|
| 플레이어 속도 | 140→210 px/s | 160→250 px/s | 160→250 px/s |
| 동시 음표 수 | 5개 | 4개 | 4개 |
| 음표 TTL | 무한 | 3.5초 | 2.8초 |
| 수간호사 속도 | 60→110 px/s | 170→290 px/s | 200→340 px/s |
| F 최대 동시 수 | 2개 | 10개 | 14개 |
| 동시 F 투척 수 | 1개 | 3개 | 4개 |
| F 투척 주기 | 3.5→2.0초 | 1.0→0.35초 | 0.8→0.25초 |
| 맵 종류 | easy맵 | hard맵 | hard맵 |
| 이교수 등장 | ❌ | ❌ | ✅ |
| 석조무사 등장 | ✅ | ✅ | ❌ |
| 목표 점수 | 60점 | 50점 | 30점 |

> 모든 속도는 시간 경과에 따라 선형 보간 (시작값 → 최대값).
> `curveT = 1 - (남은시간 / 45)`

## 6. 맵 시스템 (2종)

### easy맵
- 외곽 벽 + 중앙 기둥 1개 (2×4타일)
- 넓고 단순한 개방 공간

### hard맵 (normal·hard 공용)
- 외곽 벽
- 모서리 방 4개 (각각 문 1개)
- 중앙 기둥 + 추가 기둥들
- 복잡한 동선, 수간호사/이교수 협공 압박

### 타일 규격
- 타일 1개: 20×20pt
- 맵: 48열 × 24행 = 960×480pt (Phase 1-5 재설계 — 모바일 드론 카메라 대응)

## 7. 게임 오브젝트

### 7-1. 플레이어

| 속성 | 값 |
|---|---|
| 크기 | 16×20px (NURSE_W×NURSE_H) |
| 이동 | D-Pad 누르고 있는 동안 해당 방향으로 이동 |
| 속도 | 난이도별 baseSpeed→maxSpeed 보간 |
| 스프라이트 | 캐릭터별 픽셀 아트 (방향 4방향, 프레임 3종: idle/step1/step2) |
| 충돌 | 음표: 수집 / F·청진기: 피격 / 벽: 이동 불가 |
| 무적 | 스킬 발동 시 일부 무적 (이간호 착지 0.5초, 임간호 매혹 중) |
| 동결 | 청진기 피격 시 2초간 이동 불가 (상 난이도) |

### 7-2. 음표 (수집물)

| 속성 | 값 |
|---|---|
| 기호 | ♪ |
| 동시 최대 수 | 난이도별 4~5개 |
| TTL | easy: 무한 / normal: 3.5초 / hard: 2.8초 |
| 스폰 조건 | 플레이어·수간호사로부터 최소 4타일(맨해튼) 이상 떨어진 빈 타일 |
| 수집 효과 | 점수 증가 + 콤보 증가 + 음계 사운드 |

### 7-3. 화캉스 보너스 아이템 (변기)

| 속성 | 값 |
|---|---|
| 스폰 주기 | 12초마다 판정 |
| 스폰 확률 | 15% |
| TTL | 8초 |
| 효과 | 음표 2개 수집과 동등 (점수+2, 콤보+2) |
| 토스트 | 수집 시 "화캉스 보너스!" 텍스트 0.9초 표시 |

### 7-4. 수간호사 (메인 적)

| 속성 | 값 |
|---|---|
| 행동 | 플레이어를 향해 이동 (직선 추적) |
| 속도 | 난이도별 obsBaseSpeed→obsMaxSpeed 보간 |
| F 투척 | 주기·동시 수 난이도별 상이 (§5 참조) |
| 임간호 스킬 중 | F 대신 A 투척 (A 수집 시 점수 2배) |
| 박병장 이후 | 5초간 공포 도주 (플레이어 반대 방향) |

### 7-5. F 투사체

| 속성 | 값 |
|---|---|
| 기호 | F (빨간색) |
| 속도 | 웹 기준 projectile speed (난이도 비례) |
| 방향 | 발사 시점 플레이어 위치를 향한 직선 |
| 충돌 | 플레이어 → 즉시 게임오버 / 벽 → 소멸 |
| 임간호 스킬 중 | A로 교체됨 (수집 가능) |

### 7-6. 석조무사 (하/중 난이도 전용 NPC)

| 속성 | 값 |
|---|---|
| 행동 | 맵 4지점을 사각 순환 순찰 (투사체 없음) |
| 속도 | 55 px/s 고정 |
| 접촉 효과 | 즉시 게임오버 아님 → **AIRFORCE 이스터에그 발동** |
| 역할 | "마주치면 잡혀갑니다" 경고로 플레이어를 겁줌 → 접촉하면 사실 아군이었다는 반전 |

### 7-7. AIRFORCE 이스터에그 (석조무사 접촉 시 발동)

| 속성 | 값 |
|---|---|
| 발동 조건 | 석조무사와 플레이어 접촉 |
| 오버레이 | "나와라 박병장!" 텍스트 + 확인 버튼 |
| 비행기 | 화면 좌→우 통과 (2.4초, 48×14px) |
| 폭탄 | 오버레이 닫힘 후 300ms → 화면 플래시 420ms |
| 수간호사 효과 | 5초간 공포 도주 (플레이어 반대 방향) |
| 수간호사 복귀 후 | F 재스폰 (초기 수 × 1.0 비율) |

### 7-8. 이교수 (상 난이도 전용 NPC)

| 속성 | 값 |
|---|---|
| 행동 | 수간호사와 독립적으로 순찰 이동 |
| 속도 | 70 px/s |
| 청진기 투척 | 주기 2.5→1.4초 보간, 동시 최대 4개 |
| 청진기 속도 | 220 px/s |
| 피격 효과 | 2초간 플레이어 이동 불가 (동결) |
| 피격 토스트 | "청진기 명중!" 텍스트 1초 표시 |

## 8. 점수 & 콤보 시스템

### 점수
- 음표 1개 수집: **+1점** (기본)
- 콤보 3 이상에서 수집: **+2점** (2배)
- 화캉스 보너스: **+2점 + 콤보 +2**
- 임간호 스킬 중 A 수집: **점수 2배**

### 콤보
- 음표 수집 시 +1 증가
- 마지막 수집 후 2.5초 이내 재수집 안 하면 초기화
- 콤보 3 이상: 수집 점수 2배 적용
- HUD Combo 슬롯에 현재 콤보 표시

### 사운드 (콤보 연동)
- 음표 수집 시 C장조 스케일 음계 재생
- 콤보 단계마다 음계 상승: C4→D4→E4→G4→A4→C5→D5→E5→G5→A5
- 10콤보 초과 시 A5 고정

## 9. 컷씬 시스템

### 인트로 컷씬 (게임 시작 전)
- **제목**: "어느 한적한 병동의 오후"
- **텍스트 (하/중)**: 수간호사 순찰 틈에 작곡 노트 꺼내는 내용
- **텍스트 (상)**: 이교수 청진기 내러티브로 전환

### 중간 컷씬 1 — 15초 경과 (캐릭터별 분기)
| 캐릭터 | 속마음 텍스트 |
|---|---|
| 김간호 | "후렴구에 들어갈 코드를 아직 못 찾았어… 조금만 더!" |
| 정간호 | "빨리 만들고 산 가야 하는데… 오늘 날씨 완벽한데!" |
| 건간호 | "빨리 만들고 북클럽 가야 하는데… 이번 주 책 아직 반도 못 읽었어." |
| 임간호 | "빨리 만들고 토리 보러 가야 하는데… 우리 애기 밥시간이야." |
| 이간호 | "빨리 만들고 대만 가야 하는데… 비행기 시간이 촉박해." |

### 중간 컷씬 2 — 30초 경과
- **제목**: "수간호사의 눈초리 · 30초"
- **텍스트**: `"{캐릭터이름} 학생, 거기서 뭐 하나?" 수간호사의 F가 더 거세게 날아든다.`

### 석조무사 경고 컷씬 (하/중 게임 시작 전)
- **제목**: "경고 · 석조무사 출현"
- **텍스트**: "수간호사의 충실한 부하 석조무사가 출현합니다! 마주치면 잡혀갑니다. 절대 만나지 마세요."

### 이교수 경고 컷씬 (상 난이도 게임 시작 전)
- **제목**: "경고 · 이교수 출현"
- **텍스트**: "학교에서 나온 깐깐한 이교수가 청진기를 들고 순찰을 돕니다! 맞으면 잠시 움직일 수 없게 됩니다. 피하세요."

> 컷씬 중 게임 일시정지. "계속하기" 버튼으로 재개.

bestByChar: {

## 11. 졸업장 시스템

### 발급 조건
- 선택한 캐릭터로 하/중/상 난이도 **모두** 목표 점수 이상 달성

### 졸업장 내용
```
CERTIFICATE OF GRADUATION
실습 수료 증서

[캐릭터 픽셀 아바타]

다사다난한 실습을 마치고 {이름}는 드디어 졸업하였다.
이제 세상이라는 악보 위에 마음껏 노래를 부르며 자유롭게 살 것이다.

{최초 졸업 날짜}          hgfolio · 김간호는 음악박사 ♪
```

### 기능
- 종료 화면에서 "졸업장 보기" 버튼 → 졸업장 오버레이 표시
- "이미지로 다운로드" → 사진 앱 저장 (UIImageWriteToSavedPhotosAlbum)
- 최초 졸업 날짜는 영속 저장 (기록 갱신해도 날짜 유지)

## 13. 스프라이트 시스템

### 픽셀 캐릭터 규격
- 크기: 16×20px
- 방향: 4방향 (down / up / left / right)
- 프레임: 3종 (idle / step1 / step2)
- 팔레트: 테마(다크/라이트)에 따라 색상 변경

### 캐릭터별 시각 특징
| 캐릭터 | 특징 |
|---|---|
| 김간호 | 번머리 (올림머리) |
| 정간호 | 근육질, 곡괭이 소품 |
| 건간호 | 안경 + 책 소품 |
| 임간호 | 긴 머리 + 고양이 귀 |
| 이간호 | 단발 + 강아지 귀 |

### 수간호사 스프라이트
- 별도 픽셀 아트 (백발 안경)

### 이교수 스프라이트
- 별도 픽셀 아트 (교수 스타일)

### 석조무사 스프라이트
- 별도 픽셀 아트

## 16. 기술 구현 매핑 (웹 → iOS)

| 웹 요소 | iOS SpriteKit 구현 |
|---|---|
| Canvas 렌더링 | `SKScene` + `SKSpriteNode` (게임 노드는 `worldNode: SKNode` 자식) |
| 카메라 / 뷰포트 | **`SKCameraNode`** — 매 프레임 `player.position` 추적, 월드 경계 클램핑 |
| HUD / D-Pad / Skill 오버레이 | `SKCameraNode`의 자식 노드 (카메라 좌표계 → 항상 화면 고정) |
| 게임 루프 | `update(_ currentTime:)` + dt 보간 |
| 맵 타일 (화면보다 큰 월드) | `SKSpriteNode` 배열 또는 `SKTileMapNode` (모두 `worldNode` 자식) |
| 플레이어 | `SKSpriteNode` + 방향별 텍스처 애니메이션 (`worldNode` 자식) |
| 음표 스폰 | `SKAction.repeatForever(sequence([run, wait]))` |
| 충돌 감지 | `SKPhysicsBody` + `SKPhysicsContactDelegate` |
| D-Pad | `SKSpriteNode` 버튼 4개 (touchesBegan/Ended), 반투명, 카메라 좌표계 |
| 오버레이 (시작/캐릭터/스킬/컷씬/종료/졸업장) | `SKNode` 레이어 또는 SwiftUI `.fullScreenCover` |
| localStorage | `UserDefaults` |
| WebAudio 사인파 | `AVAudioEngine` oscillator 또는 사운드 파일 |
| 테마 전환 | `SKScene` 배경색 + 노드 색상 재적용 |
| 이미지 저장 (졸업장) | `UIImageWriteToSavedPhotosAlbum` |
| 콤보 음계 | `AVAudioEngine` 주파수 제어 |

## 17. 구현 단계 (Phase별 범위)

### Phase 1 — 플레이어 이동
- GameScene 기본 구조, D-Pad, 플레이어 이동, 벽 충돌

### Phase 2 — 핵심 게임 루프
- 음표 스폰/수집, 수간호사 AI, F 투사체, 타이머, 점수/콤보, HUD

### Phase 3 — 전체 오버레이 & 화면 흐름
- 시작/캐릭터 선택/스킬/컷씬/종료/졸업장 오버레이
- 난이도 분기, 기록 저장

### Phase 4 — 추가 NPC & 이스터에그
- 석조무사, 이교수, AIRFORCE 이스터에그
- 화캉스 보너스 아이템

### Phase 5 — 스킬 시스템
- 4캐릭터 스킬 구현 (돌진/소집/모범생/여행)

### Phase 6 — 폴리싱
- 픽셀 아트 스프라이트, 사운드, BGM, 앱 아이콘, 테마 전환

## 18. MVP 범위 재정의

**Phase 1~2 완료 = 플레이 가능한 최소 게임**

MVP에서 일시적으로 단순화:
- 스프라이트: 색깔 사각형으로 대체 (나중에 교체)
- 캐릭터: 김간호 1명 고정 (Phase 3에서 선택 추가)
- 난이도: 하(easy) 1개 고정 (Phase 3에서 추가)
- 오버레이: 간단한 시작/종료만 (Phase 3에서 완성)
- NPC: 수간호사만 (Phase 4에서 석조무사·이교수 추가)
- 스킬: 비활성 (Phase 5에서 구현)
