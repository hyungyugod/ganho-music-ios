# 게임 디자인 결정 — GanhoMusic iOS

이 문서는 **"왜 이렇게 만드는가"** 를 명문화한다.
Planner / Generator / Evaluator는 SPEC을 쓰기 전 반드시 이 문서를 참조하여, 기능 추가 시 게임 정체성과 어긋나지 않도록 한다.

핵심 원칙: **"음악박사" 라는 이름값을 코드로 증명한다.** 단순 dodge-collect가 아니라 음악적 호흡이 한 줄이라도 들어가야 한다.

---

## 0. 자전적 배경 (이 게임의 정체성)

이 게임은 단순 픽셀 아케이드가 아니라 **개발자 본인의 자전적 경험**을 게임화한 작품이다.

> 개발자(현규)는 간호학과 학생 시절 **병동 실습 중 몰래 노트에 작곡을 해서 음반을 발표**했다.
> "수간호사의 호통(F)을 피하며 음표를 모은다"는 코어 메커닉은 그 경험의 **메타포**다 — 감시자의 눈을 피해 자기만의 창작(음표=곡 작업)을 이어가는 그림.
> "음악박사"라는 이름은 *간호 실습생이지만 진짜 학위는 음악에서 따고 싶었다*는 자조 섞인 자기 표현.

### 이 사실이 디자인 결정에 미치는 영향

1. **톤은 "본인의 농담"** — 외부 IP 패러디나 일반 게임 클리셰가 아니다. 메시지 카피는 본인이 직접 쓴 것처럼 들려야 한다 (§6 톤 가이드 참조).
2. **의료/간호직 비하 농담은 절대 금지** — 본인이 속한 직군이고, 자조와 비하는 다르다.
3. **5명 캐릭터(김/정/건/임/이)는 본인의 인간관계 모티브**일 가능성이 높음 → 함부로 추가/삭제·이름 변경하지 않는다.
4. **졸업장 시스템 ([GDD.md](GDD.md) §11) 의 "다사다난한 실습을 마치고 졸업했다" 텍스트는 자전적 마무리** → 톤을 "축하 멘트"식으로 바꾸지 않는다.
5. **수간호사·이교수 같은 NPC는 적이지만 악인은 아님** — 실제 권위 캐릭터에 대한 친근한 풍자 톤. 분노·복수·잔혹 묘사 ❌.

> 메커닉을 단순화·삭제하려 할 때는 **이 항목에 자전적 의미가 있는지 먼저 확인**한다.

## 1. 코어 루프 (Core Loop)

```
[음표 발견] → [수간호사 위치 계산] → [경로 선택]
   → [수집 / 회피] → [콤보 누적] → [점수 갱신]
   → 루프 (45초간 반복)
```

**플레이어가 30초 후에도 계속 하고 싶게 만드는 동력:**

1. **콤보 시스템** — 음표를 박자(beat)에 맞춰 연속 수집할수록 배수 증가 (1× → 2× → 4×). 한 박자 놓치거나 피격 시 콤보 리셋.
2. **최고 점수 갱신 욕구** — 매 판이 끝날 때 직전 최고 점수와 비교. "+187점" 같은 차이값을 명시적으로 보여줌.
3. **45초의 인지 부하 곡선** — 0~15초 음표만, 15~30초 적 등장, 30~45초 적 빈도 2배. 끝까지 긴장감 유지.

## 2. 음악 메커닉 (Rhythm Layer)

**MVP 수준에서 한 스푼만:**

- 게임 시작과 동시에 일정 BPM의 비트(드럼 킥 또는 메트로놈) 재생.
- 음표는 **BPM과 동기화된 간격**으로 스폰 (예: 120 BPM = 0.5초 간격).
- 비트 타이밍 ±150ms 이내에 음표 수집 시 → **"On Beat" 보너스 +5점 + 콤보 +1**.
- 비트와 무관하게 수집해도 기본 점수는 들어옴 (입문자 진입장벽 낮춤).

**확장 (Phase 4 이후 옵션):**
- 곡의 후렴구에서 음표 스폰 패턴이 멜로디 라인을 그림.
- 적의 F 투사체가 비트 강박에 발사 → 패턴 학습 가능.

> 이 한 줄이 "김간호는 *음악*박사" 의 정당성을 만든다. 박자 무시한 dodge-collect는 다른 게임 1000개 중 하나에 묻힌다.

## 3. 난이도 곡선 (Difficulty Curve)

**45초 단일 세션, 3구간 점진 증가:**

| 구간 | 음표 스폰 | 적 행동 | 비트 | 인지 부하 |
|---|---|---|---|---|
| 0~15초 (도입) | 1.0초 간격 | 없음 | 100 BPM | 낮음 — 조작 익히기 |
| 15~30초 (전개) | 0.7초 간격 | 수간호사 1명, 느린 추격 | 110 BPM | 중간 — 회피 시작 |
| 30~45초 (절정) | 0.5초 간격 | 수간호사 1명 + F 투사체 가속 | 120 BPM | 높음 — 풀가동 |

**난이도는 시간 경과에 따라 자동 변화.** 별도 난이도 선택 없음 (MVP 단일 모드).

## 4. 실패 조건 / 생존 시스템

**HP 1짜리 즉사 모델 — 단, 1회 자원 소모형 보호 (Shield)**

- 기본: F 투사체 또는 수간호사 본체 접촉 시 즉시 게임 오버 (45초 끝까지 못 감).
- 게임 시작 시 보호막(Shield) 1개 보유. 첫 피격은 보호막이 흡수, 두 번째 피격에 사망.
- 음표 30개 수집할 때마다 보호막 1개 회복 (최대 1개 스택).

**왜 이 설계인가:**
- 즉사 모델은 긴장감을 만들지만 너무 가혹 → 1회 보호막으로 학습 곡선 보정.
- 보호막 회복 조건이 "음표 30개" → 수집 행동이 생존과 직접 연결, 코어 루프 강화.

**45초 풀생존 시:** 잔여 보호막 1개당 +50 보너스. 안 다치고 클리어한 플레이어를 보상.

## 5. 점수 의미 / 등급 / 기록

**점수 산식:**

```
판당 점수 = (음표 점수 × 콤보 배수)
            + (On-Beat 보너스 합계)
            + (잔여 시간 보너스)   // 게임 오버 시 0
            + (잔여 보호막 보너스)  // 풀생존 시 +50
```

**등급:**

| 등급 | 기준 점수 | 의미 |
|---|---|---|
| S | 800+ | 김간호 = 음악박사 |
| A | 500~799 | 잘 한다 |
| B | 200~499 | 무난 |
| C | 0~199 | 다음 판을 위한 준비운동 |

**최고 기록 저장**: `UserDefaults` 키 `bestScore` / `bestGrade` / `bestDate`. Phase 3 마일스톤.

**판 종료 화면:**
- 큰 점수 + 등급 + "이전 최고 대비 ±N점" 차이 표시.
- 차이가 양수면 갱신 축하 멘트 (톤 가이드 §6 참조).

## 6. 톤 / 캐릭터 / 유머 (차별화 핵심) 🎭

**김간호 캐릭터 톤:**

- 간호학과 3학년이 음악박사를 꿈꾸는 다재다능형. 시험 기간에도 비트를 만든다.
- 성격: 진지한 척하지만 허당. 자신감과 자조 사이 어딘가.
- 말투: 반말, 짧고 리드미컬. 끝맺음에 의성어 / 추임새 활용.

**메시지 카피 가이드:**

| 상황 | 평범한 카피 ❌ | 김간호 카피 ✅ |
|---|---|---|
| 게임 시작 | "Game Start" | "수업 째고 음악박사 따러 간다" |
| 음표 수집 | "+10" | "♪ 좋다" / "+10 박사학위에 한 발 더" |
| 콤보 4× | "Combo x4!" | "이게 박자감이지" |
| 보호막 사용 | "Shield Used" | "수간호사: ＂잠깐 어디가＂" |
| 게임 오버 | "Game Over" | "수간호사한테 걸렸다" |
| S 등급 | "Rank S" | "김간호 = 음악박사. 인정." |
| C 등급 | "Rank C" | "다음 판은 진심으로 가자" |

**시각 톤:**
- 8~16색 고정 픽셀 팔레트 (assets.md 참조).
- 김간호 색상 = 흰색 가운 + 청록 머리띠. 수간호사 = 진홍 가운.
- F 투사체 = 노란색 큰 알파벳 "F" 픽셀 폰트.
- 음표 = 파스텔 핑크/민트 ♪ 픽셀.

**금지 톤:**
- 의료/간호직군을 비하하는 농담 ❌
- 시험·성적에 대한 자학적 농담 (현규 본인 약점인 자의식 발동 패턴과 충돌) ❌
- 영어 슬랭 / "GG" / "Sus" 등 일반 게임 클리셰 ❌ — 김간호만의 한국어 말투로

## 7. 의도하지 않은 것 (Out of Scope, MVP)

명시적으로 **하지 않는다**. Sprint 범위 계약의 기준선:

- 멀티플레이 / 온라인 리더보드
- 인앱 결제 / 광고
- 5명 캐릭터 선택 (웹 버전 기능, MVP는 김간호 1명)
- 곡 선택 시스템
- 도전 과제 / 업적
- iPad / tvOS / macOS 지원

위 기능을 SPEC에 넣고 싶을 때는 **이 문서를 먼저 업데이트**한 뒤 SPEC을 쓴다.
