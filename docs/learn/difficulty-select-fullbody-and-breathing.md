# 난이도 화면 아바타를 풀바디로 + 글자 안 겹치게 만들기

## 한 줄 요약
난이도 고르는 화면 왼쪽 카드의 작은 얼굴 아바타를, 스킬 설명 화면처럼 **온몸이 보이는 픽셀 캐릭터**로 바꾸고, 카드와 글자가 너무 붙어 답답해 보이던 걸 **30pt 아래로 살짝 내려** 호흡을 줬어요.

## 무엇이 바뀌었나

### 1. 아바타가 머리만 → 온몸으로
**Before**: `CharacterFaceNode`라는 동그란 얼굴 그림만 나와서, 스킬 설명 화면에서 본 캐릭터(빨간 십자 간호복 입은 풀바디)랑 달라 보였어요.
**After**: 스킬 설명 화면과 **똑같은 그림 만드는 코드**를 가져와서 풀바디 픽셀 캐릭터로 통일했어요. 이제 두 화면이 같은 모습으로 이어집니다.

### 2. 글자랑 카드가 안 붙음
**Before**: 헤더 "난이도를 골라요" 글자와 카드 윗선 사이 간격이 20pt밖에 안 됨 → 답답.
**After**: 카드를 30pt 아래로 내려서 **50pt 호흡** 확보. 시선이 글자 → 카드로 자연스럽게 흐름.

### 3. 시작 버튼도 살짝 아래로
카드를 아래로 내렸더니 시작 버튼이랑 너무 가까워질 뻔해서 **시작 버튼도 따라 살짝 내렸어요**. 화면 밖으로 나가면 안 되니까 안전장치(클램프)도 달았습니다.

## 왜 이런 방식으로 했나

### Spring Boot 비유로 이해하기
Spring Boot에서 `application.yml`에 설정값(`server.port=8080` 같은)을 따로 두고 코드에서는 `@Value`로 읽어 쓰죠? SpriteKit도 똑같이 **상수(GameConfig.swift)에 숫자를 모아두고** 화면 코드에서는 그 이름을 참조해요. 직접 `80`이나 `-40` 같은 숫자를 쓰면 "매직 넘버"라 부르며 금지합니다(이유: 나중에 누가 보면 80이 뭔지 몰라서).

### V3/V4/V5 접미사가 뭔지
이전 Sprint에서 만든 상수(`OffsetY = -10`)를 **그대로 두고**, 새 값은 V5 접미사로 따로 만들었어요(`OffsetYV5 = -40`).
- **이유**: 다른 화면이 옛날 상수를 쓰고 있을 수 있어서, 값을 직접 바꾸면 그쪽이 망가질 위험이 있음.
- **Spring Boot 비유**: API v1을 그대로 두고 v2를 새로 만드는 거랑 비슷해요. 옛 클라이언트는 v1, 새 클라이언트는 v2.

### 시작 버튼 산식 — 가장 아래 것 채택 (`min`)
시작 버튼이 들어갈 자리를 3가지 후보 중 **가장 낮은 곳**(작은 y값)으로 골랐어요:
1. V5 절대 위치(`-200`)
2. 우측 난이도 카드 바닥에서 36pt 떨어진 곳
3. 좌측 요약 카드 바닥에서 36pt 떨어진 곳

그리고 화면 밖으로 안 나가게 **`max` 클램프** 하나 더 → "가장 낮은 곳, 단 화면 안".

```
buttonY = max( min(v3Y, v4RightY, v5LeftY), 화면 하단 안전선 )
```

### 코드 한 줄 변경의 위력
`baseY = midY + difficultySelectSummaryCardOffsetYV5` 이렇게 카드 중심 위치만 1줄 바꿨더니, **카드 안의 이름/얼굴/스킬/속도칩 4개가 다 같이** 30pt 내려갔어요. 왜냐하면 그 4개는 카드 중심(baseY)에서 상대 위치로 정의돼 있어서. → "**기준점 한 곳만 옮기면 자식들이 따라온다**"는 게 SpriteKit 좌표 시스템의 강점.

## 다음에 응용할 수 있는 점
- 화면 사이에 같은 캐릭터를 보여주고 싶으면 **텍스처 생성 코드를 복사**해서 통일 (참고할 코드는 SkillExplanationScene.swift Line 80~99)
- 새 화면 만들 때 노드 위치는 **컨테이너 중심 좌표 1개 + 자식 상대 OffsetY 여러 개** 패턴이 유지보수 편함
- "이 상수 값을 좀 바꿔야 하는데 다른 화면이 쓰고 있을지 몰라" → V숫자 접미사로 새 상수 만들고, 옛 상수는 그대로 두기
