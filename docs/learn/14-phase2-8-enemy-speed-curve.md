# 14 · Phase 2-8 · 수간호사가 점점 빨라진다 ⚡

> **이번 작업 한 줄**: 게임이 진행될수록 수간호사가 *점점 빨라짐*. 시작은 60(천천히), 끝(45초 후)은 110(빨리).

---

## 1. 게임이 어떻게 변하나?

**전 단계까지**: 수간호사 속도가 *항상 60*. 게임 시작이나 끝이나 똑같음.

**이번 단계**: 시간이 갈수록 *점점 빨라짐*.

```
시간:    0초 ─────────────────────── 45초
속도:   60 ━━━━━━━━━━━━━━━━━━━━━ 110
          (천천히)              (점점 빨리)
```

비유로 말하면: **마라톤에서 시간이 갈수록 코치 자전거가 빨라지는 느낌**. 처음엔 여유롭게 도망갈 수 있지만, 시간이 지날수록 *거리 벌리기 어려움*. 게임에 *긴장감*이 생김.

---

## 2. 어떻게 점점 빠르게 만드나?

### 2-1. 보간(Interpolation)이란?
**두 값 사이의 *어느 지점*인지 비율로 표현**.

예: 시작값 60, 끝값 110일 때
- *반쯤 왔으면*: 60 + (110-60) × 0.5 = 60 + 25 = **85**
- *80% 왔으면*: 60 + (110-60) × 0.8 = 60 + 40 = **100**

여기서 0.5, 0.8 같은 *비율*을 `t`라고 부름. 0이면 시작값, 1이면 끝값.

### 2-2. 시간을 비율로 바꾸기
게임 시간이 45초인데, 남은 시간이 30초면 진행률은?
- 진행률 = 1 - (남은 시간 / 전체 시간) = 1 - (30/45) = **0.33** (33%)

시작 시: 남은 시간 45 → 진행률 0
중간: 남은 시간 22.5 → 진행률 0.5
끝: 남은 시간 0 → 진행률 1

### 2-3. 합치면
```
진행률 t = 1 - 남은시간 / 45
속도 = 60 + (110 - 60) × t
```

이걸 *매 프레임* 계산해서 enemy 속도에 반영.

## 4. 새로 배운 것 (쉬운 설명)

### 4-1. **선형 보간이란?**
> 두 값 사이를 *직선처럼* 채우는 방법.

```
시작값: A
끝값:   B
비율:   t (0 ~ 1)

결과 = A + (B - A) × t
```

수학으로 보면 어렵지만, 일상에서는 *익숙*함. 예:
- 양념 1숟가락에서 2숟가락 사이의 *반*은 1.5숟가락.
- 1만원과 5만원의 *80%*는 1 + (5-1) × 0.8 = 4.2만원.

게임에서 자주 등장 — *부드럽게 변하는 값*은 거의 다 선형 보간.

### 4-2. **`curveT` 변수명**
> "곡선상의 위치 비율(t)"를 줄인 이름.

수학에서 보간을 *곡선 매개변수*라 부르고 t로 표기. 0이면 시작, 1이면 끝.

```swift
let curveT = 1.0 - remainingTime / GameConfig.gameDuration
```

(GDD에 명시된 공식 그대로)

### 4-3. **시그니처 확장 vs 추가**

이전 EnemyNode.update 시그니처:
```swift
func update(deltaTime: TimeInterval, targetPosition: CGPoint)
```

이번에 추가:
```swift
func update(deltaTime: TimeInterval, targetPosition: CGPoint, speedT: CGFloat)
```

새 인자 `speedT` 추가. 호출처(GameScene) *1곳*만 변경되니 *디폴트 값 안 줘도* OK.

> 인자가 늘어나면 호출처가 *컴파일 에러*로 알려줌. Swift의 안전 장치.

### 4-4. **수간호사가 빨라지는 게 *왜* 게임을 재밌게 만드나?**
시간이 흐를수록 *난이도 상승* → 플레이어가 *점점 더 잘 해야* 함. 게임 디자인의 기본 패턴 (페이서pacer).

처음엔 *여유롭게 학습* → 익숙해질 때쯤 *압박 시작* → 끝까지 살아남으면 *성취감*. 45초 게임에 *작은 산*이 생김.

## 8. 회고

### 8-1. 막혔던 것
**전혀 없음.** 1차 빌드 성공 + **만점 10.0/10** 합격. 변경 표면 16줄로 매우 작아 회귀 위험 0. SPEC §"기능 상세"에 코드 스켈레톤 명시 → Generator가 그대로 옮김.

### 8-2. 새로 배운 것
1. **선형 보간은 게임의 기본** — 두 값 사이를 *비율로* 채우는 방법. 속도/색/위치 등 *부드럽게 변하는 모든 값*에 적용 가능.
2. **`curveT` 변수명** — 수학에서 *곡선 매개변수*를 t로 표기. 0은 시작, 1은 끝. 게임 코드에서 자주 등장하는 컨벤션.
3. **시그니처 확장은 안전 장치** — Swift는 호출처가 *컴파일 에러*로 알려줌. 새 인자 추가가 *누락 없이* 반영됨.
4. **NaN 가드는 *항상 먼저*** — 보간 계산 *이전*에 magnitude > 0 검사. 0으로 나누는 위험 차단.
5. **시간 압박이 게임을 살린다** — 같은 속도가 영원히 유지되면 *지루*. 점점 빨라지면 *긴장감* + *마지막 순간 도전감*.

1. **Phase 2-9 — F 발사 주기 보간**: 3.5초 → 2.0초. SKAction.repeatForever를 *동적 wait*로 바꾸는 패턴 필요.
2. **Systems 폴더 분리**: GameScene이 422줄 → 300줄 미만으로.
3. **무적 시간 (Phase 5)**.
4. **F 시각 효과 (Phase 6)**.
