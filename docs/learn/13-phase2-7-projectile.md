# 13 · Phase 2-7 · 수간호사가 F를 던진다 🎯

> **이번 작업 한 줄**: 수간호사가 *3.5초마다* 노란 F를 나(김간호) 쪽으로 던진다. F에 맞으면 게임 끝. 벽에 닿으면 사라진다.

---

## 1. 게임이 어떻게 변하나?

**전 단계(2-6)까지**: 수간호사가 *직접 다가와서* 잡으러 옴. 나는 *멀리 도망*만 가면 됨.

**이번 단계(2-7)**: 수간호사가 멀리서도 **F를 던짐**. 나는 *가까운 위협(접촉)* + *먼 위협(F)* 둘 다 피해야 함.

```
지금까지                    이번 작업
👵 → → → → 🧑‍⚕️             👵 ⟶ F → → 🧑‍⚕️
(직접 다가옴)               (직접 다가옴 + F도 던짐)
```

비유로 말하면: **도지볼**. 술래가 직접 와서 잡기도 하고, *공도 던짐*. 공이 날아오는 방향을 *미리 보고* 옆으로 피해야 함.

---

## 2. F는 어떻게 동작하나?

### 2-1. 발사 시점에 위치 정해짐
F가 발사되는 *그 순간*의 내 위치를 보고 *직선*으로 날아옴.

→ 즉, **발사 후에 내가 옆으로 빠지면 F가 빈 공간으로 날아감!** 이게 회피의 핵심.

### 2-2. 벽에 닿으면 사라짐
F가 외곽 벽이나 중앙 기둥에 닿으면 *그 자리에서 사라짐*. 영원히 남지 않음.

### 2-3. 한 번에 최대 2개만
화면에 F가 *2개*까지만 떠 있을 수 있음. 2개가 다 날아다니면 *3.5초 후에도* 새로 발사 안 됨. 그래야 게임이 너무 어렵지 않음.

### 2-4. F에 맞으면 게임 끝
나(김간호)와 F가 닿으면 *즉시* 게임 종료. 시간 만료나 수간호사 접촉과 똑같음.

## 4. 새로 배운 것 (쉬운 설명)

### 4-1. **카테고리(`PhysicsCategory`)란?**
> 게임 안 모든 물체에 *번호표*를 붙이는 것.

```
플레이어 = 1번
음표    = 2번
수간호사 = 4번
벽      = 8번
F      = 16번  ← 이번에 추가!
```

번호가 1, 2, 4, 8, 16... *2배씩 커지는 이유*는 *여러 개 동시에 켜고 끄기*가 쉬워서. (스위치 5개 같은 느낌)

**Spring 비유**: `Set<RoleEnum>`처럼 *여러 역할*을 한꺼번에 가지는 거랑 비슷.

### 4-2. **반복 발사는 어떻게?**
SpriteKit이 주는 `SKAction`으로 *3.5초 기다리고 → 발사*를 *영원히 반복*.

```swift
let wait = SKAction.wait(forDuration: 3.5)
let fire = SKAction.run { [weak self] in self?.fireProjectile() }
let loop = SKAction.repeatForever(.sequence([wait, fire]))
self.run(loop, withKey: "fireProjectiles")
```

- `wait` = "기다려"
- `fire` = "발사해"
- `repeatForever` = "영원히 반복"
- `withKey: "fireProjectiles"` = 이름표. 게임 끝날 때 이 이름으로 *멈춤*.

**왜 `Timer` 안 쓰나?** 게임 일시정지하면 같이 멈춰야 자연스러운데 `Timer`는 *벽시계*라 안 멈춤. `SKAction`은 *씬 시간*이라 같이 멈춤.

### 4-3. **F는 어디로 날아가야 하나? (벡터 계산)**

수간호사 위치 = (300, 200), 나 위치 = (500, 350)이라고 치자.

```
가야 할 방향:
- x 차이 = 500 - 300 = 200 (오른쪽으로)
- y 차이 = 350 - 200 = 150 (위로)
- 거리   = √(200² + 150²) = 250
```

이걸 *거리로 나눠서 단위 벡터*(길이 1짜리 화살표)로 만든다:
```
unitX = 200 / 250 = 0.8
unitY = 150 / 250 = 0.6
```

여기에 *속도*(160)를 곱하면 매 초당 얼마나 움직일지 정해짐:
```
실제 속도 X = 0.8 × 160 = 128 (초당)
실제 속도 Y = 0.6 × 160 = 96  (초당)
```

이게 *직선 운동의 기본*. 어디든 같은 패턴.

### 4-4. **collision vs contact의 차이**
SpriteKit 물리에는 두 가지 종류 알림이 있다:

- **collision**: *물리적으로 막힘*. 벽에 부딪히면 멈추거나 튕긴다.
- **contact**: *닿았다는 알림만*. 막지는 않고 *통과*시키되 "닿았어요!"라고 알려준다.

**F의 설정**:
- `collisionBitMask = 0` → *아무것도 안 막음*. 벽도 통과.
- `contactTestBitMask = player | wall` → 플레이어와 벽에 닿으면 *알림*만.

→ 알림 받으면 *그 자리에서 F를 사라지게* 처리. 만약 collision으로 했다면 F가 *벽에 박혀 영원히 그 자리*에 남음.

이건 음표(NoteNode)에서 이미 한 번 배운 패턴. 똑같이 적용.

### 4-5. **충돌 처리 함수 분리 (didBegin)**
이전에는 `didBegin` 안에 *모든 충돌 처리*가 한 곳에 있었다. 이제 *3가지 종류*가 됨:
- 수간호사랑 닿으면 → 게임 끝
- F랑 닿으면 → 게임 끝 OR F 소멸
- 음표랑 닿으면 → 점수 추가

3가지부터는 *함수로 나누는 게 깔끔*. (3개 이상 = 분리 임계점)

```swift
func didBegin(_ contact: SKPhysicsContact) {
    let categories = bodyA.categoryBitMask | bodyB.categoryBitMask  // 두 개를 합쳐서 검사

    if categories & 수간호사 != 0 { 게임끝(); return }       // 한 줄이면 그냥 인라인
    if categories & F != 0      { F충돌처리(); return }    // 여러 줄이면 함수로
    if categories & 음표 != 0    { 음표충돌처리() }
}
```

**Spring 비유**: `@RestControllerAdvice`에서 예외 종류별로 핸들러 메서드 분리하는 거랑 같음.

### 4-6. **게임 끝날 때 F도 정지시키기**
F는 *마찰이 0*이라 한 번 출발하면 *영원히 같은 속도*. 게임 끝나도 화면에 *계속 날아다님* — 이상함.

그래서 `endGame()`에서:
```swift
worldNode.enumerateChildNodes(withName: "projectile") { node, _ in
    node.physicsBody?.velocity = .zero    // F 모두 정지
}
```

월드 안에서 이름이 "projectile"인 모든 노드를 찾아서 *속도 0*으로 만듦.

## 8. 회고

### 8-1. 막혔던 것
**없음.** 1차 빌드 성공 + 모든 회귀 테스트 통과 + 9.65/10 점수. Phase 2-6 hotfix로 *기반*이 정상화된 후 첫 큰 추가 기능이라 *깔끔하게* 들어감.

### 8-2. 새로 배운 것
1. **카테고리는 1, 2, 4, 8, 16... 2배씩** — 여러 개를 한꺼번에 켜고 끄려면 이 규칙이 필요.
2. **collision과 contact는 다르다** — 막을 거냐 vs 알림만 받을 거냐. 게임에서 자주 등장.
3. **함수 분리는 *분기 3개부터*** — 2개까지는 if/else가 단순. 3개부터는 함수가 깔끔.
4. **마찰 0인 물체는 영원히 같은 속도** — 게임 끝낼 때 *명시적으로* 멈춰야 함.
5. **Spring과 비교**: `@Scheduled(fixedRate=3500)`처럼 주기 작업. 다만 게임은 *씬 시간* 기반이라 일시정지에 자동 동기.

### 8-3. 다음으로 미룬 것
1. **Phase 2-8 — 시간 따라 점점 빨라지기**: 수간호사 속도(60→110), F 발사 주기(3.5→2.0초)가 게임이 진행될수록 *점점 빨라짐*.
2. **Systems 폴더 분리**: GameScene이 422줄로 너무 김. SpawnSystem / ContactRouter로 나눠야.
3. **무적 시간 (Phase 5)**: 이간호 스킬 "대만여행" 0.5초 무적.
4. **F 시각 효과 (Phase 6)**: 회전 / 잔상 / 소리.
