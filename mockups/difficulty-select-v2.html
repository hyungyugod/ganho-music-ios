이 화면의 핵심 결정 (Sprint 6 신규)

📍 왜 이 화면이 마지막인가

RPG에서 "직업도 모르고 난이도부터 정하기"는 부자연스럽다. 캐릭터·스킬을 본 뒤 "그래서 이 친구로 얼마나 도전할래?"가 인지 순서. 결정 비용을 흐름 뒤로 미루기.

👤 캐릭터 요약 카드 (좌)

방금 본 캐릭터·스킬을 한 번 더 노출해서 "어떤 친구로 무슨 스킬 갖고 가는지" 잊지 않게. 미니 아바타 + 코랄 이름 뱃지 + 스킬명 + 민트 속도 칩 4가지 정보 자급자족.

🎚️ 난이도 3장 카드 (우)

쉬움(50초·민트) · 보통(45초·골드) · 어려움(35초·코랄). 색 점으로 톤만 분리, 시각 노이즈 없이 정보 위계. 선택 시 떠오르기(translateY -6 + scale 1.05).

🧭 브레드크럼 [난이도] 강조

"캐릭터 · 스킬 · [난이도]" 우상단 칩. 난이도만 코랄 뱃지 — 사용자가 지금 어디 있는지 즉시 인지. 5단계의 마지막임을 시각으로 약속.

▶ 시작 버튼만 = 명확한 마무리

다른 화면이 두 버튼(뒤로 + 다음)인 것과 다르게 여기는 시작 하나만 정가운데. 0/6px 입체 그림자 + 흰 화살표 chip — "이게 마지막 결정"이라는 시각 약속.

↩️ 백 = 분기별 다른 텍스트(.kim)

.kim은 스킬 화면을 스킵하므로 백버튼이 `← 캐릭터 다시`. 그 외 4캐릭터는 `← 스킬 다시`. 같은 위치, 다른 텍스트 — 분기를 디자인이 자연 표현.

SpriteKit 구현 매핑

좌측 카드

SKShapeNode(200×260, cornerRadius 22) + CharacterFaceNode(scale 0.65, zPos 105) + 코랄 이름 뱃지(SKShapeNode + SKLabelNode) + Gowun Dodum 스킬명 + ganhoScrubMint α 0.4 속도 칩.

우측 3장

DifficultyCardNode 기존 컴포넌트 그대로 재사용 — 폭 88/간격 12 보존. setSelected에서 spring overshoot + 링 글로우 자동.

상단 바

GlassPillNode 백버튼 + DarkContextChipNode 브레드크럼. 두 컴포넌트는 Sprint 1에서 만든 그대로 — 외부 좌표·텍스트만 분기.
